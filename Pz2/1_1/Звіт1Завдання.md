# Звіт про розробку: Програма форматування Excel

## 1. Як ми це зробили і що використовували
В ході роботи була написана програма мовою **Java 11**, яка відкриває файли Excel, перевіряє вміст комірок і змінює колір їхнього фону залежно від того, що всередині (цифри, літери або і те, і інше).

**Використані технології та інструменти:**
- **Apache POI (v5.2.5)** — потужна бібліотека для роботи з форматами Microsoft Office (зокрема, `.xls` та `.xlsx`) з мови Java.
- **RegEx (Регулярні вирази)** — вбудований у Java механізм для пошуку збігів у тексті. Використовувався для визначення наявності латинських чи кирилічних (українських) літер (`[a-zA-Zа-яА-ЯіІїЇєЄґҐ]`) та цифр (`\d`).
- **Apache Maven** — система автоматичної збірки проєкту. Вона управляє життєвим циклом та автоматично завантажує потрібні бібліотеки (POI API та Log4j) з інтернету. Ми використали вбудований переносний Maven, щоб не встановлювати його глобально в операційну систему.

---

## 2. Опис головних файлів

- **`pom.xml`**: Головний конфігураційний файл Maven. У ньому описано версію Java (11), а також список зовнішніх залежностей (Apache POI Core, POI OOXML та Log4j). Без цього файлу Java (та Maven) не знала б, звідки підтягувати класи для роботи з Excel.
- **`src/main/java/com/example/ExcelFormatter.java`**: Основний файл з логікою програми. У ньому написаний метод `main`, який виконує всю роботу.
- **`exel_files/`**: Папка (директорія), яку програма використовує як джерело (вхідні дані). Звідси читаються всі файли `.xls` та `.xlsx`.
- **`exel_changed/`**: Папка, куди програма зберігає нові відформатовані файли, додаючи до їхньої оригінальної назви суфікс `_changed` (наприклад, `Book1_changed.xlsx`).

## 3. Детальний розбір коду (`ExcelFormatter.java`)

```java
File inputDir = new File("exel_files");
File outputDir = new File("exel_changed");

if (!inputDir.exists()) {
    System.out.println("Директорія 'exel_files' не існує або не знайдена.");
    return;
}
if (!outputDir.exists()) {
    outputDir.mkdirs();
}
```
Створюємо об'єкти `File`, які вказують на папки входу та виходу. Якщо папка джерел `exel_files` не існує, програма завершує роботу — далі робити нічого. Якщо папки для збереження результатів `exel_changed` ще немає — вона автоматично створюється за допомогою методу `.mkdirs()`.

```java
File[] files = inputDir.listFiles((dir, name) -> name.endsWith(".xlsx") || name.endsWith(".xls"));
if (files == null || files.length == 0) { ... }
```
Отримуємо список усіх файлів у папці `exel_files`. Лямбда-функція працює як фільтр та відбирає тільки ті файли, розширення яких є форматом Excel: `.xlsx` або `.xls`.

```java
Pattern letterPattern = Pattern.compile("[a-zA-Zа-яА-ЯіІїЇєЄґҐ]");
Pattern digitPattern = Pattern.compile("\\d");
```
Створюємо два шаблони (регулярні вирази) і попередньо компілюємо їх перед циклом задля пришвидшення роботи. `letterPattern` означає: знайти будь-яку латинську чи кирилічну літеру. `digitPattern` означає: знайти будь-яку цифру (від 0 до 9).

```java
CellStyle styleOnlyNumber = workbook.createCellStyle();
styleOnlyNumber.setFillForegroundColor(IndexedColors.GREEN.getIndex());
styleOnlyNumber.setFillPattern(FillPatternType.SOLID_FOREGROUND);
```
Створюємо об'єкт `CellStyle` (стиль комірки Excel). Потім задаємо для цього стилю зелений колір фону (`GREEN`) як основний. Властивість `FILL_PATTERN_SOLID` (або `SOLID_FOREGROUND`) вказує на те, що заливка комірки має бути суцільною, без текстур та візерунків. Подібним чином створено синій, жовтий та помаранчевий стилі.

```java
DataFormatter dataFormatter = new DataFormatter();
```
Ініціалізуємо клас `DataFormatter`. Це дуже корисний інструмент Apache POI. Він бере на себе найскладнішу частину: перетворює вміст будь-якої комірки у рядок (String) точно так само, як сам Excel показує її на екрані, незалежно від типу (текст, формула, дата чи число).

```java
for (Sheet sheet : workbook) {
    for (Row row : sheet) {
        for (Cell cell : row) {
```
Використовуємо вкладені цикли `for-each` для того, щоб пройтися всією ієрархією документа: по кожному аркушу (`Sheet`), потім по кожному заповненому рядку (`Row`), і, нарешті, по кожній комірці (`Cell`) всередині цього рядка.

```java
String cellValue = dataFormatter.formatCellValue(cell);
                    
boolean hasLetter = letterPattern.matcher(cellValue).find();
boolean hasDigit = digitPattern.matcher(cellValue).find();
```
Використовуючи наш `DataFormatter`, зчитуємо видимий текст комірки в локальну змінну `cellValue`. Далі метод `matcher(...).find()` шукає наші регулярні вирази в цьому тексті. Параметр `hasLetter` стане дорівнювати `true`, якщо знайшлась хоча б одна потрібна літера. `hasDigit` — якщо зустрічається хоча б одна цифра.

```java
if (hasDigit && !hasLetter) {
    cell.setCellStyle(styleOnlyNumber);
} else if (!hasDigit && hasLetter) {
    cell.setCellStyle(styleOnlyLetter);
} else if (hasDigit && hasLetter) {
    cell.setCellStyle(styleBoth);
} else {
    cell.setCellStyle(styleNeither);
}
```
Застосовуємо логіку зафарбовування фону комірок через звичайні умовні оператори `if-else`:
1. Якщо є цифра, але немає жодної букви — встановлюємо зелений фон комірки (`styleOnlyNumber`).
2. Якщо в комірці наявні тільки літери (цифр немає) — встановлюємо синій фон (`styleOnlyLetter`).
3. Якщо є рядок, що містить і літери, і цифри — встановлюємо жовтий фон (`styleBoth`).
4. У будь-якому іншому випадку (порожня комірка, або комірка тільки з пробілами чи спецсимволами) — помаранчевий фон.

```java
String origName = file.getName();
int dotIndex = origName.lastIndexOf('.');
String newName = (dotIndex == -1) 
        ? origName + "_changed" 
        : origName.substring(0, dotIndex) + "_changed" + origName.substring(dotIndex);
                
File outputFile = new File(outputDir, newName);
```
Алгоритм модифікації імені збереженого файлу. Ми знаходимо останню позицію крапки (роздільника між назвою і розширенням, наприклад, `.xlsx`). Розрізаємо ім'я на дві частини та вставляємо суфікс `_changed` прямо перед нею. Потім на основі роздільника створюємо підготовлений шлях у папці `exel_changed`.

```java
try (FileOutputStream fos = new FileOutputStream(outputFile)) {
    workbook.write(fos);
}
```
Створюємо файл для збереження його на жорсткому диску. Робимо це за допомогою безпечного макросу `try-with-resources`. Метод `workbook.write(fos)` серіалізує і записує всі внесені зміни з пам'яті комп'ютера прямо в новостворений Excel файл, після чого потік запису `fos` автоматично закривається без жодного ризику витоку пам'яті.
