# Звіт про розробку програми для пінг-моніторингу

## 1) Що ми використовували, стек
* **Мова програмування:** Java (Java SE)
* **Підхід програмування:** Об'єктно-орієнтоване програмування (ООП)
* **Патерни проєктування (Design Patterns):**
  * **Singleton (Одинак):** для єдиної точки доступу до файлу з налаштуваннями (`config.properties`).
  * **Strategy (Стратегія):** для гнучкості генерації звітів без зміни основного коду (окремі класи для TXT, XML, HTML).
  * **Factory (Фабрика):** для створення потрібної стратегії звіту на основі вибору користувача.
* **Підхід до тестування/даних:** Data-Driven (дані та налаштування винесені в зовнішні файли `hosts.txt` та `config.properties`).

## 2) Які технології/бібліотеки
Проєкт побудовано виключно на стандартних бібліотеках Java (Java Standard Edition API), без використання сторонніх фреймворків чи залежностей (no third-party dependencies):
* `java.net.InetAddress` — для здійснення мережевих запитів та перевірки доступності вузла (`isReachable()`).
* `java.io.*` (`FileInputStream`, `FileReader`, `FileWriter`, `BufferedReader`) — стандартний пакет вводу-виводу для читання та запису текстових і конфігураційних файлів.
* `java.util.*` (`Properties`, `List`, `ArrayList`, `Scanner`) — для парсингу файлу властивостей, збереження наборів даних у колекції та взаємодії з користувачем через консоль.

## 3) Опис коду пінгування та налаштування проєкту

```properties
ping.interval.ms=1000
ping.count=4
ping.timeout.ms=3000
```
**Опис:** Файл `config.properties` містить ключові налаштування моніторингу: інтервал між запитами (мс), кількість пінгів для кожного хоста та час очікування відповіді (таймаут). Це реалізує підхід ізоляції даних від логіки.

```text
google.com
8.8.8.8
1.1.1.1
ukr.net
192.168.0.1
```
**Опис:** Файл `hosts.txt` містить список URL та IP-адрес для перевірки. Винесення списку в окремий зовнішній файл (Data-Driven) дозволяє легко змінювати цілі для моніторингу без втручання в Java код.

```java
import java.io.FileInputStream;
import java.util.Properties;

public class ConfigManager {
    private static ConfigManager instance;
    private Properties properties;

    private ConfigManager() {
        properties = new Properties();
        try (FileInputStream fis = new FileInputStream("config.properties")) {
            properties.load(fis);
        } catch (Exception e) {}
    }

    public static synchronized ConfigManager getInstance() {
        if (instance == null) instance = new ConfigManager();
        return instance;
    }
    // методи getInterval(), getPingCount()...
}
```
**Опис:** `ConfigManager.java`. Клас реалізує патерн Одинак (Singleton). Приватний конструктор зчитує `config.properties`, а метод `getInstance()` гарантує, що конфігурація завантажується лише один раз і доступна глобально з будь-якого місця програми.

```java
public class PingResult {
```

## 4) Опис коду для збереження в різних форматах

### Загальний інтерфейс та Фабрика

```java
public interface ReportStrategy {
    void generateReport(List<PingResult> results, String fileName);
}
```
**Опис:** `ReportStrategy.java` визначає загальний інтерфейс для складання звітів. Це реалізація патерну Стратегія (Strategy). Дозволяє програмі використовувати різні алгоритми збереження (TXT, XML, HTML), не змінюючи основний код додатку.

```java
public class ReportFactory {
    public static ReportStrategy getReportStrategy(String reportType) {
        switch (reportType.toUpperCase()) {
            case "XML": return new XmlReportStrategy();
            case "HTML": return new HtmlReportStrategy();
            case "TXT": default: return new TxtReportStrategy();
        }
    }
}
```
**Опис:** `ReportFactory.java` реалізує патерн Фабрика (Factory). Його задача — інкапсулювати умовну логіку створення конкретного об'єкта звіту. Він приймає рядок формату і віддає ініціалізований об'єкт потрібної стратегії.

### Стратегії форматів звітів

```java
public class TxtReportStrategy implements ReportStrategy {
    @Override
    public void generateReport(List<PingResult> results, String fileName) {
        try (FileWriter writer = new FileWriter(fileName + ".txt")) {
            for (PingResult result : results) {
                writer.write(String.format("Хост: %s | Доступний: %b | Сер. час: %.2f мс\n", result.getHost(), result.isReachable(), result.getAverageTimeMs()));
            }
        } catch (IOException e) {}
    }
}
```
**Опис:** `TxtReportStrategy.java` — стратегія для формату звичайного тексту. Створює файл з розширенням `.txt` і за допомогою `FileWriter` записує дані рядок за рядком у легкому для сприйняття людиною консольному/текстовому стилі.

```java
public class XmlReportStrategy implements ReportStrategy {
    @Override
    public void generateReport(List<PingResult> results, String fileName) {
        try (FileWriter writer = new FileWriter(fileName + ".xml")) {
            writer.write("<?xml version=\"1.0\" encoding=\"UTF-8\"?>\n<PingResults>\n");
            for (PingResult result : results) {
                writer.write("  <Result>\n    <Host>" + result.getHost() + "</Host>\n...");
            }
            writer.write("</PingResults>\n");
        } catch (IOException e) {}
    }
}
```
**Опис:** `XmlReportStrategy.java` — стратегія створення XML-файлу (`.xml`). Вона будує деревоподібну структуру тегів `<PingResults>` та `<Result>`. Ідеально підходить, якщо ці результати потрібно автоматично експортувати або парсити іншими програмами.

```java
public class HtmlReportStrategy implements ReportStrategy {
    @Override
    public void generateReport(List<PingResult> results, String fileName) {
        try (FileWriter writer = new FileWriter(fileName + ".html")) {
            writer.write("<!DOCTYPE html><html><body><table border=\"1\">\n");
            for (PingResult result : results) {
                String color = result.isReachable() ? "green" : "red";
                writer.write("<tr><td>" + result.getHost() + "</td><td style=\"color:" + color + ";\">...");
            }
            writer.write("</table></body></html>");
        } catch (IOException e) {}
    }
}
```
**Опис:** `HtmlReportStrategy.java` — генерує звіт у вигляді веб-сторінки (`.html`). Використовує стандартні HTML-теги для створення табличника (`<table>`, `<tr>`, `<td>`). Додатково містить базові CSS-стилі (зелений або червоний колір шрифту), щоб користувачу було візуально легко швидко оцінити доступність хостів просто відкривши файл у браузері.
